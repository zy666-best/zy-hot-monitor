# 内容过滤与评分规则

本文档定义热点监控技能在完成多源搜索后，如何对原始结果进行去重、过滤和质量评分，以便提炼出真正有价值的热点话题。

---

## 第一阶段：基础清洗

在进入任何过滤逻辑前，先对每条原始结果做标准化处理：

1. **清除 HTML 标签**：将 `<em>`, `<b>`, `<strong>` 等标签从标题和摘要中删除
2. **折叠空白**：将连续空格、换行、制表符替换为单个空格
3. **提取来源域名**：从 URL 中提取主域名（去掉 `www.` 前缀），例如 `www.36kr.com` → `36kr.com`
4. **清除追踪参数**：从 URL 中删除 `utm_source`、`utm_medium`、`spm`、`from` 等参数

---

## 第二阶段：硬过滤（直接丢弃）

以下情况的结果**直接丢弃**，不进入后续评分：

| 过滤条件 | 原因 |
|----------|------|
| 标题为空或长度 < 8 字符 | 无实质内容 |
| 摘要为空且标题 < 15 字符 | 信息量太少 |
| 标题是纯数字或纯符号 | 非内容结果 |
| URL 为空或不是有效 HTTP/HTTPS 链接 | 无法溯源 |
| 标题和摘要中没有任何与领域相关的词 | 完全无关 |

**无关内容判断方法**：
- 将领域名和搜索变体词拆分为若干词元
- 检查标题+摘要中是否包含至少一个词元
- 如果完全不包含任何词元，且来源不是可信域名，则丢弃

---

## 第三阶段：去重合并

相同或高度相似的内容只保留一条，优先保留信息更丰富的版本。

### URL 去重
- 去掉 URL 末尾的 `#fragment` 和追踪参数后，若两条结果 URL 完全相同，合并为一条

### 标题去重
对标题进行归一化后比较：
1. 转换为小写
2. 删除所有标点符号和特殊字符
3. 折叠空白

若归一化后的标题**前 60 个字符**相同，则视为重复，合并为一条。

### 合并策略
同一话题多源命中时：
- 保留摘要更长的版本
- 累计 `crossSourceCount`（多源命中数），每多一个来源 +1
- 保留最早的发布时间
- 记录所有命中的搜索引擎名称

---

## 第四阶段：质量评分

对每条去重后的结果计算质量分（满分约 100 分），高于阈值的才进入热点提炼。

### 评分维度

#### 来源类型分（固定加分）

| 来源类型 | 分值 | 判断条件 |
|----------|------|----------|
| 新闻（news） | +28 | sourceEngine 含 "news"，或来自新闻类网站 |
| 普通网页（web） | +18 | 其他网页内容 |
| 社交/视频（social） | +8 | bilibili.com, weibo.com 等 |

#### 可信域名加分

以下域名的内容额外加 **+20 分**：

```
openai.com, anthropic.com, googleblog.com, microsoft.com,
github.com, huggingface.co, techcrunch.com, theverge.com,
venturebeat.com, wired.com, arstechnica.com,
bilibili.com, weibo.com, 36kr.com, sspai.com,
jiqizhixin.com, qbitai.com
```

低可信来源扣分（-18 分）：URL 含 `baijiahao.`、`sohu.com/a/`、`163.com/dy/article`、`toutiao.com`

#### 内容长度分（摘要字数）

| 摘要长度 | 分值 |
|----------|------|
| ≥ 180 字符 | +8 |
| ≥ 100 字符 | +6 |
| ≥ 50 字符 | +4 |
| ≥ 25 字符 | +2 |
| < 25 字符 | -4 |

#### 多源命中加分

每多一个来源额外命中，加 8 分，上限 +18 分（即 3 个以上来源时封顶）：
```
跨源加分 = min(18, (crossSourceCount - 1) × 8)
```

#### Bilibili 播放量加分（仅 social 类型）

| 播放量 | 加分 |
|--------|------|
| ≥ 10,000 | +7 |
| ≥ 1,000 | +3 |
| 点赞 ≥ 20 | +5 |
| 点赞 ≥ 5 | +3 |

#### 语言加分

| 语言 | 加分 |
|------|------|
| 中英混合（mixed） | +4 |
| 纯中文或纯英文 | +2 |

### 最终过滤阈值

**保留 score ≥ 40 的结果**进入热点提炼阶段。

---

## 第五阶段：热点聚类

评分通过后，对剩余结果进行话题聚类，将同一事件的不同报道合并为一个热点话题。

### 聚类判断
若两条结果满足以下任一条件，视为同一话题：
- 标题的核心实体词（人名、产品名、公司名）完全重合
- 摘要中的核心动词短语高度相似（如"发布"、"融资"、"宣布"等相同）
- 一条内容是另一条的转载或转述（URL 域名不同但核心内容一致）

### 聚类后处理
- 同一话题选摘要最丰富的那条作为代表
- `crossSourceCount` 加上被合并的条目数
- 热点最终排序分 = `crossSourceCount × 0.4 + qualityScore × 0.6`

---

## 最终输出列表规范

过滤和聚类完成后，输出 **Top 5-8 热点话题**，每条包含：

```
{
  "rank": 1,
  "title": "热点话题标题",
  "summary": "50-150字的分析摘要，说明为何是热点、核心关注点是什么",
  "url": "最佳来源链接",
  "source": "来源域名",
  "sourceEngine": "搜索引擎名",
  "crossSourceCount": 3,
  "qualityScore": 72,
  "publishedAt": "发布时间（若可获取）"
}
```

若所有结果质量分均低于 40，则降低阈值至 25，并在报告中注明"本次监控数据质量有限，以下结果仅供参考"。
