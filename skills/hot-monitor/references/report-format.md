# 报告输出格式

本文档定义热点监控报告的完整输出模板和格式规范。

---

## 标准报告模板

```markdown
# {领域名称} 热点监控报告

**监控时间**：{YYYY-MM-DD HH:mm}  
**搜索来源**：{实际命中的搜索源，用 / 分隔，例如 DuckDuckGo / Bing News / Bilibili}  
**原始候选**：{原始收集条数} 条 → 过滤后 {保留条数} 条 → 提炼 {最终条数} 个热点

---

## 🔥 热点话题

### {rank}. {热点标题}

{1-2 句分析摘要，说明为何是热点，核心争议或关注点是什么，尽量客观具体}

**来源**：{来源域名}｜**引擎**：{搜索引擎名}｜**多源命中**：{crossSourceCount} 个来源  
🔗 {来源链接}

---

（重复上述结构，列出所有 Top 热点）

---

## 📊 趋势摘要

{2-4 句话总结本次监控发现的整体趋势：核心热门方向、主要声音或争议、值得持续关注的信号}

---

*本报告由热点监控技能自动生成，基于 DuckDuckGo、Bing、Bilibili 等免费搜索源，不含付费数据源。*
```

---

## 格式规范

### 标题
- 报告标题格式：`# {领域名} 热点监控报告`，领域名直接使用用户输入的原始词
- 热点序号从 1 开始，格式为 `### 1. 标题`

### 分析摘要写作规范
- **长度**：50-120 字，不超过 150 字
- **内容要求**：
  - 说明话题的**核心事件或观点**（是什么）
  - 说明**为何成为热点**（为什么重要，行业影响、争议点、时效性等）
  - 避免空洞描述，如"这是一个重要的新闻"、"引发广泛关注"等无信息量的表述
- **来源忠实**：摘要内容必须来自实际搜索到的原文，不可捏造细节

### 来源信息格式
```
**来源**：36kr.com｜**引擎**：bing-news｜**多源命中**：3 个来源
🔗 https://36kr.com/p/xxxxx
```

- 若 URL 过长（> 80 字符），保持完整 URL，不截断
- 若无发布时间可提取，省略时间字段，不显示"未知"

### 趋势摘要写作规范
- 不要逐一重复已列出的热点内容
- 聚焦"整体趋势"：本期热点集中在哪个子方向？有哪些新的信号？哪些话题预计会持续发酵？
- 若热点数量少于 3 条（搜索结果不足），在趋势摘要中说明原因（如"当前该领域热点信号较弱，建议扩大搜索周期"）

## 特殊情况处理

### 情况 1：搜索结果不足（< 5 条通过过滤）

```markdown
# {领域名} 热点监控报告

**监控时间**：{时间}  
**搜索来源**：{实际使用的源}

⚠️ 本次搜索获取的有效内容较少（{N} 条），可能原因：
- 当前该领域热点信号较弱
- 部分搜索源暂时无法访问

以下为现有结果，质量供参考：

（按常规格式列出所有通过过滤的结果，即使不足 5 条）

---

**建议**：可尝试更换具体的子领域词进行搜索。
```

### 情况 2：某个搜索源完全失败

在报告的"搜索来源"行中注明：
```
**搜索来源**：DuckDuckGo / Bing News（Bilibili 暂时不可访问，已跳过）
```

### 情况 3：所有搜索源均失败

```markdown
# {领域名} 热点监控报告

❌ 本次监控未能获取搜索结果，所有搜索源均无法访问。

可能原因：网络连接问题或搜索源暂时不可用。

**建议**：请检查网络连接后重试，或稍后再次运行热点监控。
```
